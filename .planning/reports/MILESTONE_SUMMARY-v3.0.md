## 1. Overview
Milestone v3.0 focuses on **Production Scaling & Resilience** for the supportLegal system. Following the successful implementation of the frontend and chat interface in v2.0, this milestone aims to optimize performance, reduce latency, and ensure the system can handle production loads efficiently. The primary focus so far has been on database communication optimization and caching.

## 2. Architecture & Design
The architecture has been enhanced with the following components:
- **Semantic Cache**: A Qdrant-based cache (`semantic_cache` collection) stores query vectors and corresponding LLM responses. This reduces redundant LLM calls for similar queries.
- **gRPC Migration**: Communication with the Qdrant database has been switched from REST to gRPC (port 6334) to reduce latency.
- **Payload Pre-filtering**: Added indexing and filtering on fields like `nam_ban_hanh` and `linh_vuc` to narrow down search space before vector search.
- **Background TTL Cleanup**: A background worker script handles cleaning up cache entries older than 30 days.

## 4. Key Decisions & Rationale
| Decision | Rationale |
|----------|-----------|
| **Qdrant for Semantic Cache** | Avoided adding Redis to the stack to keep infrastructure simple and leverage existing Qdrant capabilities. |
| **gRPC for Qdrant** | Lower latency compared to REST, critical for real-time RAG performance. |
| **Random UUIDs for Cache Points** | Simplifies ID generation for cache entries in Qdrant. |
